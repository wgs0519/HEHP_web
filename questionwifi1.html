WIFI 1 문제

코드 설명:

이 코드는 ESP32를 사용하여 Wi-Fi 액세스 포인트를 생성하고, 클라이언트가 접속하면 캡티브 포털 페이지를 표시하는 예제입니다.

  • WiFi.softAP(ssid, password): ESP32를 액세스 포인트(AP) 모드로 설정하여 지정된 SSID와 비밀번호를 사용해 네트워크를 생성한다.
  • WiFi.softAPConfig(apIP, apIP, IPAddress(255, 255, 255, 0)): AP 모드에서 ESP32의 IP 주소와 서브넷 마스크를 설정합니다. 이 코드에서는 IP 주소를 192.168.4.1로 설정한다.
  • dnsServer.start(DNS_PORT, "*", apIP): DNS 서버를 시작하여 모든 도메인 이름을 AP의 IP 주소로 리다이렉트합니다. 클라이언트가 어떤 웹사이트를 요청하더라도 ESP32의 IP로 연결된다.
  • webServer.on("/", handleRoot): 웹 서버에서 기본 경로("/")로 들어오는 요청을 처리하기 위해 handleRoot() 함수를 등록합니다. 이 함수는 캡티브 포털 페이지를 제공한다.
  • webServer.on("/submit", HTTP_POST, handleSubmit): 클라이언트가 폼을 제출하면 해당 데이터를 처리하기 위해 handleSubmit() 함수를 호출합니다. 이메일과 비밀번호를 수집하여 시리얼 모니터에 출력한다.
  • webServer.onNotFound(handleNotFound): 클라이언트가 잘못된 URL을 요청할 때 모든 요청을 캡티브 포털 페이지로 리다이렉트하기 위해 handleNotFound() 함수를 호출한다.
  • dnsServer.processNextRequest(): DNS 서버가 들어오는 요청을 처리하고, 클라이언트를 올바른 웹 페이지로 리다이렉트한다..
  • webServer.handleClient(): 웹 서버가 클라이언트의 HTTP 요청을 처리하고 적절한 응답을 보냅니다.
  • 이 코드 전체는 "캡티브 포털"을 구현하여 사용자가 지정된 네트워크에 연결될 때마다 특정 웹 페이지를 표시합니다. 이 웹 페이지에서 사용자는 이메일 주소와 비밀번호를 입력할 수 있으며, 입력된 정보는 시리얼 모니터에 출력된다.
  • 이를 통해 임의의 사용자의 정보를 강탈하는 것이 가능하다.
